Syntax Overview

A tour of Nexus syntax — generics, pattern matching, memory semantics, and more.

Comments

Nexus has two comment styles. Block comments use /* … */ and are used for inline annotations. Inline documentation comments use /! … !/ and appear on declarations to describe their purpose.

comment styles
/* This is a block comment. It ends automatically at the next newline
   unless you open a new comment. */

/! This is a doc comment — appears on functions, classes, methods. !/

public Add(T value) -> LinkedList<T>   /! returns self for chaining !/

Primitive Types

Nexus is statically typed. All primitives have explicit sizes. There is no implicit numeric coercion.

I/O & String Interpolation

Nexus provides two print functions. Printf evaluates interpolation expressions inside { } at runtime. Print outputs the string literally — no interpolation.

printf vs print
Printf(" 5 /  2 = {5/2}  \n");  /* 5 /  2 = 2.5  */
Printf(" 5 // 2 = {5//2} \n");  /* 5 // 2 = 2     */
Printf(" 5 %  2 = {5%2}  \n");  /* 5 %  2 = 1     */

Print(" 5 /  2 = {5/2}" );   /* 5 /  2 = {5/2}   (literal) */
Print(" 5 // 2 = {5//2}");   /* 5 // 2 = {5//2}  (literal) */
Note: Anything inside { } within a Printf string is a full expression — you can call functions, do math, and access fields there.

Variables & Assignment

Variables are declared with their type. Nexus distinguishes between copy assignment (=) and explicit move (<-), giving you control over ownership at every assignment site.

copy vs move
i32 x = 10;               /* copy */
i32 y <- x;              /* explicit move — x is no longer valid */

Options<LinkedListNode<T>> tmp &= self.head;  /* borrow reference */
SyntaxSemantics
T x = valueCopy assignment
T x <- valueExplicit move (transfers ownership)
T x &= valueBorrow (reference, not ownership)

Sum Types (Enums)

Sum types are declared with the sum keyword. Each variant can optionally carry data. They are the foundation for safe null-free programming in Nexus.

Options<T> — built-in sum type
sum Options<T>
{
    None,
    Some(T value)
}

Variants with data use a constructor-like syntax. Options is the standard way to represent nullable values — there is no null keyword in Nexus.

Pattern Matching

The match expression exhaustively matches against sum type variants. Each arm uses => to bind and execute. Binding inner values uses parenthesis syntax.

match on Options<T>
match self.tail
{
    Options.Some(existingTail) =>
    {
        existingTail.next = Options.Some(node);
        node.previous = Options.Some(existingTail);
        self.tail = Options.Some(node);
    }
    Options.None =>
    {
        self.head = Options.Some(node);
        self.tail = Options.Some(node);
    }
}
Exhaustiveness: The compiler enforces that all variants of a sum type are handled. Forgetting a case is a compile error.

Classes

Classes are declared with the class keyword and a visibility modifier. Constructors are defined with the special Constructor method name. The self keyword refers to the current instance.

class declaration
private class LinkedListNode<T>
{
    public Options<LinkedListNode<T>> previous;
    public Options<LinkedListNode<T>> next;
    public T value;

    public Constructor(T value)
    {
        self.value    <- value;   /* explicit move */
        self.previous = Options.None;
        self.next     = Options.None;
    }
}

Generics

Generics use angle-bracket syntax <T> on both type declarations and method signatures. Generic parameters are resolved at compile time with no runtime overhead.

generic class + method
public class LinkedList<T>
{
    public Add(T value) -> LinkedList<T>
    {
        /! returns self for chaining !/
        /* … */
        return self;
    }

    public Get(i32 i) -> Options<LinkedListNode<T>>
    {
        /* … */
    }
}

Function Return Types

Return types are declared with -> after the parameter list. Methods that return self enable fluent chaining.

method signatures
public GetSize() -> i32
{
    return self.size;
}

public GetHead() -> Options<LinkedListNode<T>>
{
    return self.head;
}

Full Example — LinkedList

The following is a complete doubly-linked list implementation demonstrating generics, sum types, pattern matching, move semantics, and method chaining together.

linked list + main
sum Options<T>
{
    None,
    Some(T value)
}

private class LinkedListNode<T>
{
    public Options<LinkedListNode<T>> previous;
    public Options<LinkedListNode<T>> next;
    public T value;

    public Constructor(T value)
    {
        self.value    <- value;
        self.previous = Options.None;
        self.next     = Options.None;
    }
}

public class LinkedList<T>
{
    private Options<LinkedListNode<T>> head;
    private Options<LinkedListNode<T>> tail;
    private i32 size;

    public Constructor()
    {
        self.head = Options.None;
        self.tail = Options.None;
        self.size = 0;
    }

    public Add(T value) -> LinkedList<T>  /! returns self for chaining !/
    {
        LinkedListNode<T> node = new LinkedListNode<T>(value);
        match self.tail
        {
            Options.Some(existingTail) =>
            {
                existingTail.next = Options.Some(node);
                node.previous = Options.Some(existingTail);
                self.tail = Options.Some(node);
            }
            Options.None =>
            {
                self.head = Options.Some(node);
                self.tail = Options.Some(node);
            }
        }
        self.size++;
        return self;
    }

    public Get(i32 i) -> Options<LinkedListNode<T>>
    {
        Options<LinkedListNode<T>> tmp &= self.head;
        for (i32 cpt = 0; cpt < i; cpt++)
        {
            match tmp
            {
                Options.Some(Head) =>
                {
                    match tmp.next
                    {
                        Options.Some(node) => { tmp &= tmp.next; }
                        Options.None => { return Options.None; }
                    }
                }
                Options.None => { return Options.None; }
            }
        }
        return tmp;
    }

    public GetSize() -> i32 { return self.size; }
}

Main()
{
    Printf(" 5 /  2 = {5/2}  \n");  /* 5 /  2 = 2.5  */
    Printf(" 5 // 2 = {5//2} \n");  /* 5 // 2 = 2    */
    Printf(" 5 %  2 = {5%2}  \n");  /* 5 %  2 = 1    */

    LinkedList<BankAccount> list = new LinkedList<BankAccount>();
    list.Add(new BankAccount())
        .Add(new BankAccount());  /* chained */

    while true => Printf("Testing single line loops");
}
